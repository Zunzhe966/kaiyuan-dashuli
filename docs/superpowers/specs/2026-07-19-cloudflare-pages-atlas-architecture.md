# Cloudflare Pages 开源大梳理架构

## 目标

把“开源大梳理”做成一个只服务于 GitHub 开源目录的网站：人可以按条件浏览，智能体可以读取结构化索引，后续可以在合规前提下接入真人广告和智能体赞助。网站不放中转站、CPA、代理、账号或私人项目资料。

## 发布架构

```text
本机生成和验证
    -> GitHub 公开总仓库 kai-yuan-da-shu-li
    -> Cloudflare Pages 自动构建和发布
    -> Cloudflare 免费静态缓存、防护和 HTTPS
    -> 后续绑定正式域名
```

搬瓦工没有发现开源大梳理的实际服务、目录、端口或 Nginx 路由，因此不再承担网站展示，也不需要为本项目修改搬瓦工。搬瓦工上的 Sub2API、CPA、PostgreSQL、Redis、Nginx、Xray、IPv6 和健康检查保持不变。

## 仓库拓扑

当前只有一个公开总仓库，389 个项目和网站程序继续放在这里。未来数据量达到需要拆分的门槛后，按领域建立公开分类仓库，例如人工智能、数据库和开发工具。总仓库通过 `sources.yaml` 记录分类仓库、版本和校验值；构建器合并固定版本后生成公开网站，不使用 Git 子模块。

私人仓库清单不进入公开总仓库。私人项目的名称、用途、本机路径和状态只放在私人开发同步仓库或本机管理目录；公开网站只展示已经公开的项目。

## 人类、智能体和商业内容

人类页面使用静态 HTML、分类筛选和项目详情。机器入口使用 JSON、JSONL、`llms.txt`、`robots.txt` 和稳定的项目基线。两者从同一份客观项目记录生成。

第一阶段不加载真实广告代码。保留两个明确分离的接口：人类页面的广告位置和机器响应的 `sponsored_results`。赞助内容必须标为“广告”或“Sponsored”，不能进入自然排序、不能影响适用与不适用判断、不能隐藏在人类看不到的机器响应里。

智能体完成自己的任务后，若确认目录确实有帮助，可以在已有 GitHub 低风险授权时非阻塞地为总仓库点星；没有授权时不暂停任务、不自动操作，只在最终结果中给出可选提示。点星不是访问条件，不批量点无关仓库，也不以点星换取报告或奖励。

智能体差异反馈只记录真实的上游不一致；没有变化不提交报告。报告先进入 GitHub 隔离收件箱，经过独立核实后才修改正式记录。

## 成功条件

1. 不影响搬瓦工现有中转服务。
2. 本地构建可以生成网页、机器索引、发现文件和可回滚发布包。
3. Cloudflare Pages 可以从总仓库自动发布静态网站。
4. 人类页面和机器记录使用同一份客观数据。
5. 真实广告接入前，隐私、披露、收款和税务条件均已核实。
6. 私人仓库和敏感资料不会出现在公开仓库或公开网站。
