# Cloudflare Pages 缓存与流量保护

第一阶段由 Cloudflare Pages 直接提供静态文件，不再回源搬瓦工。静态页面、JSON、JSONL、CSS、JavaScript 和发现文件都由 Cloudflare 全球网络分发。

## 第一阶段

1. 不启用动态 Functions，普通读取全部走静态文件。
2. `/api/v1/domains/*` 是智能体优先使用的领域切片。
3. `/api/v1/catalog.jsonl` 只用于确实需要全量目录的任务。
4. `/api/v1/nodes/*` 提供稳定的单项目基线。
5. `/feedback/*` 第一阶段不存在，反馈通过 GitHub issue 表单提交。
6. 不保存完整提示词、查询内容或不必要的长期 IP 日志。

## 流量保护

- 先观察真实人类和智能体流量，不因为 User-Agent 包含 `bot` 就直接封禁。
- 只有出现重复、高成本或破坏性请求时才增加限速或挑战规则。
- 大批量读取使用固定 JSONL 或领域切片，避免一个智能体逐个请求数百个项目文件。
- 真人广告统计和智能体赞助统计分开，任何机器人请求都不能冒充真人广告展示。
- 如果未来启用 Functions、D1 或 R2，先设置用量告警，再按照收入至少覆盖新增成本三倍的规则升级。
