# Cloudflare 缓存与流量保护

这些规则只在域名和公开部署获得用户确认后配置。按以下顺序应用，前面的规则优先：

1. `/feedback/*` 绕过缓存。第一版不提供该 HTTP 路径，先保留边界。
2. `/api/v1/nodes/*` 与 `/api/v1/domains/*` 在边缘缓存 24 小时。
3. `/assets/*`、版本化数据和预压缩文件在边缘缓存 30 天。
4. HTML 在边缘缓存 15 分钟，并允许源站短时不可用时继续提供旧内容。
5. 尊重源站 ETag 和内容压缩，不在边缘重复改写 JSON/JSONL。
6. `POST`、带 `Authorization` 或 Cookie 的请求永不缓存。静态目录第一版只接受 GET/HEAD。

## 限速原则

- 先用 Cloudflare 缓存吸收正常抓取，不让同一个目录请求重复打到搬瓦工。
- 公开初期设置保守规则：单个来源短时间异常高频访问时返回 429；不要因为 User-Agent 写着“bot”就默认封禁。
- 观察真实智能体流量后再调阈值。搜索引擎验证过的爬虫、稳定的 JSONL 顺序下载和普通浏览必须分开观察。
- 服务器源站仅监听回环地址；公网不能绕过 Cloudflare 或现有前门直接访问 8088。
- Cloudflare 和源站日志只保留排障所需的最短时间，不长期保存搜索词、查询参数或完整访问轨迹。

限速目标是保护 1GB 服务器，不是制造虚假访问。任何广告统计都只使用广告平台认可的真人展示和点击。
